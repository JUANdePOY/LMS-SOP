# LMS-SOP Hostinger Deployment Guide

## Prerequisites
- Hostinger hosting account with PHP/MySQL support
- Node.js 18+ installed on your hosting plan (or use a VPS/Node.js hosting)
- phpMyAdmin access for database management

## Database Setup (phpMyAdmin)

1. Log in to phpMyAdmin (remote MySQL)
2. Create a new database named `lms_sop`
3. Create a MySQL user with access to the database
4. Note the connection details:
   - Host: `srv1980.hstgr.io` (or your Hostinger remote MySQL host)
   - User: `u591572634_reservist`
   - Password: your database password
   - Port: `3306`

## Environment Configuration

1. Copy the Hostinger environment file:
   ```
   npm run hostinger:setup
   ```

2. Or manually create `.env` and `server/.env` with your Hostinger credentials:
   ```
   DB_HOST=srv1980.hstgr.io
   DB_USER=u591572634_reservist
   DB_PASSWORD=your_password
   DB_NAME=lms_sop
   DB_PORT=3306
   JWT_SECRET=your_random_secret
   CORS_ORIGINS=https://yourdomain.com
   NODE_ENV=production
   ```

## Deployment Steps

### Option 1: Using the deploy script
```bash
npm run hostinger:deploy
```

### Option 2: Manual deployment
```bash
# 1. Install dependencies
npm install
cd client && npm install && npm run build && cd ..
cd server && npm install --production && cd ..

# 2. Seed demo data (optional)
npm run seed

# 3. Start the server
npm run hostinger:start
```

## Important Notes

- The `server/.env` file contains the actual database credentials — **never commit this file**
- The `.env.example` and `server/.env.example` files are safe to commit
- Remote MySQL on Hostinger may have connection timeout issues — the config includes `connectTimeout: 30000` and `reconnect: true`
- CORS is configured to allow `.hostingersite.com` domains
- The `.htaccess` file proxies API requests (`/api/*`) to the Node.js server on port 5000
- Static files are served from `client/dist/`

## Troubleshooting

### Database connection fails
- Verify the remote MySQL host is correct (check Hostinger's phpMyAdmin for the remote MySQL host)
- Ensure your IP is whitelisted in Hostinger's MySQL remote access settings
- Check that the database user has permission to connect from your server's IP

### API routes return 404
- Ensure the `.htaccess` file is in the document root
- Verify the Node.js server is running on port 5000
- Check that `client/dist/index.html` exists (build completed successfully)

### CORS errors
- Add your domain to `CORS_ORIGINS` in `.env`
- The server already allows `.hostingersite.com` and `.vercel.app` domains
