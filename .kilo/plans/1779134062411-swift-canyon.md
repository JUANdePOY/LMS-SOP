# Plan for ExternalForm Squadron Slot Limits

## Goal
Modify ExternalForm to allow selecting squadrons and setting slot limits per squadron (instead of selecting specific reservists as in InternalForm).

## Changes

### 1. Create New Component: `SquadronSlotLimits.jsx`
- Similar to `SquadronParticipantBlocks.jsx` but:
  - Replace reservist multi-select with a number input for slot limit per squadron.
  - Each block contains: `squadronId`, `squadronName`, `slotLimit` (number).
  - Props: `blocks` (array), `onChange` (function), `disabled` (boolean).

### 2. Update `TrainingForm.jsx`
#### State
- Add `squadronSlotLimits` to `externalForm` state (array of squadron limit objects).
- Initialize from training data when editing (assumes backend returns `squadron_limits` field).

#### UI
- In `ExternalFields`, replace the general `Capacity` field with `<SquadronSlotLimits />`.
- Pass `externalForm.squadronSlotLimits` as `blocks` prop.
- Use `handleSquadronSlotChange` for `onChange` prop.

#### Handlers
- Add `handleSquadronSlotChange(key, value)` to update `squadronSlotLimits` in `externalForm` state.

#### Submission
- For external training payload:
  - Remove `capacity` field.
  - Add `squadron_limits: externalForm.squadronSlotLimits.map(b => ({ squadron_id: Number(b.squadronId), slot_limit: b.slotLimit }))`.
  - (Adjust field name as per backend expectations.)

#### Validation
- Ensure at least one squadron is selected and slot limits are positive numbers (if required).

## Notes
- Backend must accept `squadron_limits` array for external trainings.
- InternalForm remains unchanged.
