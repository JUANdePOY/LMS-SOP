# Plan: Implement Registration Duplicate Prevention for External Trainings

## Problem
Users can currently register multiple times for the same external training, creating duplicate registrations in the database. This leads to inaccurate slot counts and potential issues with attendance tracking.

## Solution
Add a backend check in the `registerExternalParticipant` function to prevent registration if a user (identified by their squadron) has already registered for the same training.

## Implementation Details

### Backend Changes
**File:** `server/services/trainingsService.js`
**Function:** `registerExternalParticipant`

Add a duplicate check before inserting a new registration:
1. Extract the squadron_id from the participant data (already parsed earlier in the function for slot limit checking)
2. Query the `training_registrations` table to check for existing registrations with:
   - Same `training_id`
   - Same squadron_id in the participant_data (using JSON_UNQUOTE(JSON_EXTRACT(participant_data, "$.squadron_id")))
3. If an existing registration is found, throw an error with status code 400

### Frontend Changes
No changes are strictly required as the backend will reject duplicate attempts and the existing error handling in `RegistrationModal.jsx` will display the error to the user.

However, for improved user experience, consider:
- The frontend already handles backend errors via `submitError` state, so users will see a clear message when attempting duplicate registration
- No additional API calls or state changes are needed

### Error Message
The backend will throw an error with message: "You are already registered for this training"
This will be caught and displayed in the RegistrationModal's submitError state.

### Implementation Steps
1. Open `server/services/trainingsService.js`
2. Locate the `registerExternalParticipant` function
3. After the participant data parsing and squadronId extraction (around line 590), add the duplicate check query
4. If duplicate found, throw appropriate error before proceeding with insertion
5. Test by attempting to register for the same training twice - second attempt should show error

### Dependencies
- None: Uses existing database connection and query patterns
- Leverages existing participant data structure (squadron_id field)
