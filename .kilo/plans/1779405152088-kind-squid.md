# Plan: Add Search Bar and Filtering to Announcements Page

## Objective
Implement search and filtering functionality for the Announcements page to allow users to quickly find announcements by text search and filter by type, priority, and status.

## Current State Analysis
- **Announcements.jsx**: Displays announcements in a grid, loads data via `useAnnouncements` hook
- **Announcement data structure**: `{ id, title, type, priority, status, author, body, created_at }`
- **Available fields for filtering**: title, type (General/Training/Deployment/Administrative/Emergency), priority (low/medium/high/critical), status (active/inactive)
- **UI Pattern**: Uses Tailwind CSS, lucide-react icons, custom Button component

### 3. Update Announcements.jsx Page
**File**: `client/src/pages/Announcements.jsx`

- Add `filteredAnnouncements` computed from filters applied to `announcements`
- Add `<AnnouncementFilters />` component in header section
- Display filter results count

### 4. Filtering Logic
```javascript
const filteredAnnouncements = announcements.filter(announcement => {
  const matchesSearch = searchTerm === '' || 
    announcement.title.toLowerCase().includes(searchTerm.toLowerCase()) ||
    announcement.body.toLowerCase().includes(searchTerm.toLowerCase()) ||
    announcement.author.toLowerCase().includes(searchTerm.toLowerCase());
  
  const matchesType = typeFilter === 'all' || announcement.type === typeFilter;
  const matchesPriority = priorityFilter === 'all' || announcement.priority === priorityFilter;
  const matchesStatus = statusFilter === 'all' || announcement.status === statusFilter;
  
  return matchesSearch && matchesType && matchesPriority && matchesStatus;
});
```
