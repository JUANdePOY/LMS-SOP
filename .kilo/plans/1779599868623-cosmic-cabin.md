# Plan: Add Attachments Sections to Training Modals

## Problem Statement
Modify the current interface to include sections for displaying attachments associated with trainings (both internal and external) within modal windows. The functionality should be view-only, omitting download buttons or options.

## Current Implementation Analysis
1. **Internal Training Flow**:
   - `InternalTrainingCard.jsx` -> `TrainingDetailsModal.jsx`
   - Service: `getInternalTrainingById()` in `trainingsService.js`

2. **External Training Flow**:
   - `ExternalTrainingCard.jsx` -> `RegistrationModal.jsx`
   - Service: `getExternalTrainingById()` in `trainingsService.js`

3. **Attachment Services Available**:
   - `downloadInternalAttachment()`
   - `downloadExternalAttachment()`
   - `uploadLetterOrder()`
   - `uploadExternalLetterOrder()`
   - **Missing**: Functions to list/get attachments for a training

## Proposed Solution

### 1. Backend API Assessment
First, need to verify if attachment listing endpoints exist:
- Check if `/trainings/internal/{id}/attachments` and `/trainings/external/{id}/attachments` endpoints exist
- If not, they need to be implemented in the backend

### 2. Frontend Modifications

#### A. Internal Training Modal (`TrainingDetailsModal.jsx`)
Add attachments section after participant information:
```jsx
{/* Attachments Section */}
<div className="space-y-4">
  <p className="text-sm font-semibold text-neutral-500 dark:text-neutral-400">
    Attachments
  </p>
  {loadingAttachments ? (
    <p className="mt-1 text-neutral-600 dark:text-neutral-400">
      Loading attachments...
    </p>
  ) : attachmentError ? (
    <p className="mt-1 text-xs text-red-500">{attachmentError}</p>
  ) : (attachments?.length || 0) === 0 ? (
    <p className="mt-1 text-neutral-600 dark:text-neutral-400">
      No attachments available.
    </p>
  ) : (
    <div className="space-y-2">
      {attachments.map((attachment) => (
        <div key={attachment.id} className="flex items-center gap-3 p-2 rounded-border border-neutral-100 bg-neutral-50 dark:border-neutral-800 dark:bg-neutral-900">
          <div className="flex-shrink-0">
            {/* File type icon based on extension */}
            <AttachmentIcon fileType={attachment.file_type || attachment.name.split('.').pop()} />
          </div>
          <div className="flex-1 space-y-1">
            <p className="text-sm font-medium text-neutral-900 dark:text-neutral-100 truncate" title={attachment.name}>
              {attachment.name}
            </p>
            <p className="text-xs text-neutral-500 dark:text-neutral-400">
              {attachment.file_size ? formatFileSize(attachment.file_size) : ''}
              {attachment.uploaded_at ? ` • ${formatDate(attachment.uploaded_at)}` : ''}
            </p>
          </div>
        </div>
      ))}
    </div>
  )}
</div>
```

### 3. Service Layer Enhancements
Add new functions to `trainingsService.js`:
```javascript
getInternalTrainingAttachments: async (trainingId) => {
  try {
    const response = await api.get(`/trainings/internal/${trainingId}/attachments`);
    const body = response.data;
    return {
      success: body.success !== false,
      message: body.message,
      data: body.data || [],
    };
  } catch (error) {
    return {
      success: false,
      message: error.response?.data?.message || 'Failed to fetch attachments'
    };
  }
},

getExternalTrainingAttachments: async (trainingId) => {
  try {
    const response = await api.get(`/trainings/external/${trainingId}/attachments`);
    const body = response.data;
    return {
      success: body.success !== false,
      message: body.message,
      data: body.data || [],
    };
  } catch (error) {
    return {
      success: false,
      message: error.response?.data?.message || 'Failed to fetch attachments'
    };
  }
}
```

### 4. Component Implementation Details

#### State Management
- Add `attachments`, `loadingAttachments`, `attachmentError` state variables
- Use `useEffect` to fetch attachments when training ID changes or modal opens

#### Helper Functions
- `formatFileSize(bytes)`: Converts bytes to human-readable format (KB, MB, etc.)
- `formatDate(dateString)`: Formats date for display
- `AttachmentIcon`: Component that returns appropriate icon based on file type

#### UI/UX Considerations
- Use consistent styling with existing modal components
- Show file type icons (PDF, DOC, ZIP, etc.) 
- Display file size and upload date
- Truncate long filenames with tooltip on hover
- Maintain view-only constraint (no download/action buttons)
- Responsive layout within modal constraints

### 5. Alternative Approaches

#### Approach A: Extend Existing Data
If attachment data is already included in training objects from `getInternalTrainingById`/`getExternalTrainingById`:
- Skip additional API calls
- Directly map over `training.attachments` array
- Simpler implementation but depends on backend already providing this data

#### Approach B: Unified Attachments Modal
Instead of inline sections:
- Add "View Attachments" button in modal
- Opens separate modal focused solely on attachments
- Better for trainings with many attachments
- More complex navigation but cleaner primary modal

#### Approach C: Collapsible Section
Make attachments section initially collapsed:
- Saves space in modal
- Users expand when interested
- Good balance of accessibility and space efficiency

- Ensure no interactive elements that would initiate downloads
- If backend returns download URLs, do not render them as clickable elements
