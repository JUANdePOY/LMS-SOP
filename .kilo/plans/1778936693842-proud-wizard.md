# Plan: Improve `TrainingFilters.jsx` and integrate it into `Trainings.jsx`

## Diagnosis

### What exists now

| Artifact | Status | Issue |
|---|---|---|
| `TrainingFilters.jsx` | **Dead code** — never imported anywhere | Standalone orphan; no page uses it |
| `useTrainingFilters.js` | **Dead code** — never imported anywhere | Stale leftover hook; not wired to any component |
| `Trainings.jsx` inline filter bar (lines 247–303) | Active — self-contained | Works but is not DRY; no structured status/date-range filtering |
| `INTERNAL_STATUSES` (`trainingModel.js:4`) | `draft / published / ongoing / completed / cancelled` | Correctly surfaced in Trainings.jsx filter |
| `EXTERNAL_STATUSES` (`externalTrainingModel.js:4`) | `draft / open / closed / completed` | Not surfaced in any filter control yet |

### Problems in the abandoned `TrainingFilters.jsx`

1. **Dead code** — zero imports; excluded from the bundle tree.
2. **`onChange` used as event listener** on `<form>` — fires on every keystroke; semantically should be `onSubmit`.
3. **Status values `planned / active` don't exist anywhere in the backend** — the only shared status across both tables is `draft / completed`; `published, ongoing, cancelled` are internal-only and `open, closed` are external-only.
4. **Single date input, no end date** — "Date Range" label is misleading; cannot form a real range in a single `<input type="date">`; neither the internal nor external model has any date-range filtering today.
5. **Processed on every keystroke** in `Trainings.jsx` line 258 (`onChange={(e) => { setSearch(e.target.value); setPage(1); }}`).
6. **Status dropdown filters both internal + external on one value** — some statuses (e.g. `published`, `open`) are exclusive to one source type only.

---

## Goals

1. Resurrect `TrainingFilters.jsx` as a clean, self-contained, reusable component and import it into `Trainings.jsx`.
2. Fix the backend support gap for date-range filtering (internal trainings only — `start_datetime` + `end_datetime` columns already exist).
3. Merge internal/external status option sets so users see only valid values for the currently selected source.
4. Fix debouncing and event-handling issues in both the component and `Trainings.jsx`.
5. Minimal code changes only — no rewrites.

### Step 1 — `server/models/trainingModel.js` — Add date-range filter to `findInternalMany`

Currently `findInternalMany` takes `{ status, type, search, page, limit }`. Add `startDate` and `endDate` parameters with the same timestamp conversion applied in `resolveInternalRange`:

```js
// After the `search` block (~line 86), before ORDER BY
if (startDate) {
  sql += ' AND t.start_datetime >= ?';
  params.push(trainingModel.toDatetime(startDate, false));
```

### Step 3 — Rewrite `TrainingFilters.jsx` (targeted replacement)

Key changes only:

```jsx
// Filter status options — separate sets for each source
const INTERNAL_STATUSES = [
  { value: 'all', label: 'All Status' },
  { value: 'draft', label: 'Draft' },
  { value: 'published', label: 'Published' },
  { value: 'ongoing', label: 'Ongoing' },
  { value: 'completed', label: 'Completed' },
  { value: 'cancelled', label: 'Cancelled' },
];

const EXTERNAL_STATUSES = [
  { value: 'all', label: 'All Status' },
  { value: 'draft', label: 'Draft' },
  { value: 'open', label: 'Open' },
  { value: 'closed', label: 'Closed' },
  { value: 'completed', label: 'Completed' },
];

// Props: accept source alongside filters
const TrainingFilters = ({ source, filters, onChange, onReset }) => {
  const statusOptions = source === 'external' ? EXTERNAL_STATUSES : INTERNAL_STATUSES;

  // Controlled <form> with onSubmit — prevents accidental fires on every keystroke
  const handleSubmit = (e) => {
    e.preventDefault();
    const fd = new FormData(e.target);
    const updates = Object.fromEntries(fd);
    onChange({ ...filters, ...updates }, true); // true = user explicitly submitted
  };

  return (
    <div className="training-filters">
      <form onSubmit={handleSubmit} className="filters-form">
        {/* Search — controlled input, submit on Enter */}
        <div className="filter-group">
          <label>Search</label>
          <input name="search" defaultValue={filters.search} placeholder="Search trainings..." />
        </div>
        {/* Status — options switched by source */}
        <div className="filter-group">
          <label>Status</label>
          <select name="status" defaultValue={filters.status || 'all'}>
            {statusOptions.map(s => <option key={s.value} value={s.value}>{s.label}</option>)}
          </select>
        </div>
        {/* Date Range — two inputs; passed as startDate / endDate */}
        <div className="filter-group">
          <label>From</label>
          <input type="date" name="startDate" defaultValue={filters.startDate || ''} />
        </div>
        <div className="filter-group">
          <label>To</label>
          <input type="date" name="endDate" defaultValue={filters.endDate || ''} />
        </div>
        <div className="filters-actions">
          <button type="button" onClick={onReset} className="btn-reset">Reset</button>
          <button type="submit" className="btn-primary">Apply Filters</button>
        </div>
      </form>
    </div>
  );
};
```
